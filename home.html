What are Data Structures?


An Array, a basic data structure, storing a group of integer values

Data structures are specific formats used to efficiently store data while allowing for various operations to be performed on said data. Some examples of commonly used data structures include arrays, linked lists, stacks, and queues. Each data structure functions differently from the rest and comes with its own advantages and disadvantages. As a result, some data structures are more suited for specific tasks than others. It is up to the programmer to accurately discern which one is most appropriate for their needs.

Why are they Important?


The ability to choose the most suitable data structure for a task is essential to becoming a quality software developer. Data structures are the building blocks of a program, and choosing the wrong ones will cause said program to run extremely inefficiently. Let's say, for example, you have a collection of unordered integer values, and you wish to find the location of a specific one. If you store that data in a structure unsuitable for searching, such as an array, you would need go through it blindly, one value at a time, until you find your target. In the worst scenario, you may have to check every value in the array before you succeed. This is highly inefficient and can take a very long time if you were working with a large amount of data. However, if you store your data in a structure that's suitable for performing searches, such as a binary search tree, fewer elements will need to be checked before you find the target. Consequently, it will take less time to complete the search, making your program run much faster.

No matter which field of computer science you wish to pursue, data structures are a fundamental aspect of programming that cannot be ignored.

The 4 Basic Functions


The advantages and disadvantages of a data structure can be determined by how efficiently it can perform the following 4 functions:


  • Access: Given the location of an element within the structure, return that element's value.

  • Search: Find a target element within the structure and return its location. If the element cannot be found, return a value that reflects this (usually -1).

  • Insert: Add an element to the structure at a specific location within it.

  • Delete: Remove an existing element in the structure given its location.

We are interested the time complexities of these functions, or how their running times change as the structure's size increases. Will it take longer for these functions to run if the structure contains more data? If so, how much longer? The answers to these questions will reveal what tasks the data structure are suitable and unsuitable for.
