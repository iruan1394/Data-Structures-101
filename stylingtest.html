What is an Array?


The Array is a very simple data structure. It is a container that holds a fixed number of elements that are of the same type. Each element within the array has its own index, or position within the array, starting from 0. The first element will be at index 0, the second index 1, the third 2, and so on. Thus, an array of 5 elements will store them in indexes from 0 to 4.


Indexes of an array that holds 5 elements

Arrays also store their contents in contiguous memory. This means the elements of the array are all stored adjacent to each other in a computer's memory. If for example, the first element of an array is stored at memory location 200, the second element will be stored at memory location 201, the third at 202, and so on.

What are the Advantages of an Array?


• Fast Accessing at any Index

Because the contents of an array are stored in contiguous memory, the memory location of each element within the it is known. Therefore, it is possible to directly access an element at any index without going through previous elements. Furthermore, the access will always take the same amount of time regardless of the target index. This means that no matter how large the array may be, the time it takes to access any element within it will not change. This quality is known as random access, and very few data structures besides the array possesses it.


• Fast Traversal

Since the elements of an array are stored side by side in memory, it is quite easy to move from one element to the next. This, along with random access, makes arrays ideal for quickly going through and reading a set of data.

What are the Disadvantages of an Array?


• Fixed Size

Once an array has been created, the max amount of elements that it can hold is set in stone. When that limit is reached, it will be impossible to add more to the array unless an element is removed. Thus, it would be unwise to use arrays to store data of unknown size.


• Fixed Memory Allocation

An array will always take up a set amount of memory space no matter how many elements it contains. Even when it's empty, it will take up the same amount of memory as when it's full. Therefore, memory will be "wasted" if the array isn't fully populated. This can be avoided by simply knowing the size of the data that's going to be stored and creating the array with a matching capacity.


• Inefficient Insertion and Deletion

When an element is inserted into an array at an index that's not the last, the element that was previously at that index, as well as all of the elements after it, needs to be shifted forward. This can be extremely cumbersome if the array holds a large number of elements or if the index being inserted into is near the beginning of the array. The same holds true for deletion; all of the elements positioned after the index needs to be shifted backwards to fill in the gap.

Array Applications


Arrays are very versatile and can be used in many situations. However, they are most suitable for data that has a known size. They are also the preferred choice when a lot of accessing needs to be done on various locations within the data. Some examples of applications of the array data structure are:


  • Matrixes: A matrix in mathematics is a 2-dimensional set of numbers. It has a defined amount of rows and columns, which means it's size is fixed. This makes a nested array, or an array whose elements are also arrays, suitable for storing this data.
  • Implementation of Other Data Structures: Arrays can be used to create more complex data structures such as stacks and queues. Because of how simple arrays are, they are also extremely versatile and often become the foundations for many classes, objects, and structures in applications.
