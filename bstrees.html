What is a Binary Search Tree?


A binary search tree is part of a family of data structures known as trees. Trees, like linked lists, store data as a collection of containers called nodes. However, the nodes of a tree are completely different from those of a linked list. Tree nodes are organized using parent-child relationships, where nodes points down to one or more other nodes. A node pointing down is known as a parent while the nodes it is pointing down to are its children. Nodes that share the same parent are known as siblings. The root of a tree is the node located at its very top, thus it cannot have any parents. The leaves of a tree are the nodes that do not have any children. A subtree of a node is a tree made up of one of its children and all of that child's descendants. The height of a tree is the distance from the root node to the bottom-most leaf node.


A basic tree with a height of 3



In order for a tree to be considered a binary search tree, several criteria must be met:


  • A parent node can only have at most 2 children: a left child and a right child.

  • The left subtree of a node can only contain values less said node's.

  • The right subtree of a node can only contain values greater than said node's.

  • Any subtree within the tree must also be a binary search tree.
A binary search tree of integers (Wikipedia)

A binary search tree can either be balanced or unbalanced. In a balanced binary search tree, the heights of the subtrees of any node must not differ by more than 1. This ensures that the overall height of the binary search tree is kept to a minimum, which allows for faster searching. In an unbalanced binary search tree, a node can have subtrees with wildly different heights. In the worse case scenario, each parent may only have 1 child node leading to a tree that essentially functions as a linked list. This is known as a degenerate tree and is highly undesirable.

What are the Advantages of a Binary Search Tree?


• Efficient Search

Since the elements of a binary search tree are stored in a particular order, there is no need to blindy check each node until the target is found. Instead, we can navigate through the tree starting from the root by comparing the node we are currently on to our target. If our target's value is equal to the current node's, we have found it. If it is less than, we move to the current node's left child. If it is greater than, we move to the right child. If a situation occurs where our current node does not exist, then the tree does not contain our target. Even though this method is faster than blind checks, we may still need to go through many nodes if the tree is unbalanced or worse, degenerate.

Binary Search Tree performing a search

• Efficient Insertion and Deletion

When a node is inserted into a binary search tree, it will always be inserted as a child of an existing leaf. The only work that needs to be done is calculating which leaf the node should be inserted below and altering that leaf to point to the inserted node. The former action is done by traversing the tree in the same manner used to perform searches. The speed at which this is done depends on the height of the tree and how unbalanced the tree is. The latter action can be done instantly regardless of the tree's size or structure. All that needs to be done is altering the target leaf so that it points to the newly inserted node


As for deletion, the actions needed to be taken depends on the its status of the target node. If it is a leaf, then all that needs to be done is altering its parent so that it no longer points to the target. If the target node is a parent with a single child, then that child must replace it. If it is a parent with 2 children, then it must be replaced by the smallest node in its right subtree. The time needed to find this node depends on the height of the subtree and how unbalanced the subtree is. The actual replacement can be done instantly regardless of the tree's size and structure.

What are the Disadvantages of a Binary Search Tree?


• Balancing

A binary search tree must be balanced in order to attain optimal performance. However, even if a tree starts off as balanced, it may degenerate as nodes are added and removed. This problem can be avoided by using a self-balancing binary search tree, which maintains its balanced property by rearranging the tree as nodes are added and removed. However, such rearrangements may take up a lot of time if there are many nodes within the tree. Thus it is not recommended to use binary search trees to store large amounts of data.


• High Maintenance

When a binary search tree performs even basic functions such as insertion and deletion, many steps need to be taken to ensure that it still remains a binary search tree. As such, binary search trees require more computational space to complete tasks than other data structures.



Binary Search Tree Applications


Binary search tree's are mainly used to keep a set of data sorted as elements are added and removed. They are also used when searching needs to be performed frequently. Some applications of a binary search tree include:


  • Implementing Various Search Algorithms: There are several ways to search for an item through a set of data, but the most efficient ones require the data be sorted in a particular order beforehand. After all, if the data is underordered, the only possible method would be to blindly check each element. A binary search trees automatically sorts its contents as new nodes are added, making it an appropriate for showing how various searching algorithms work.
