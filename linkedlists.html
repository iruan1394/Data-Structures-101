What is a Linked List?


A linked list is a series of containers known as nodes. Each node normally contains 2 things, a stored data value and the memory location of the next node in the list. It can be said that each node "points" to the next. The first node in a linked list is known as the head node, and it serves as a representation of the linked list. This is because the head node is all that's needed to find every other node in the list. The last is known as the tail node. Since there is nothing which comes after the tail node, the location that it points to is null.


Linked list with 4 nodes, each one storing an integer

A standard linked list is known as a singly linked list since it can only be traversed in a single direction, forward. However, there exists a special type of linked list known as a doubly-linked list. In this linked list, each node stores not only the memory location of the next node, but that of the previous as well. This allows for backwards traversal, a function that cannot be performed by a singly linked list.


Doubly linked list with 4 nodes, each one storing an integer

A linked list can also be a circular linked list where the tail node points back to the head node as its next. Furthermore, in a circular doubly linked list, the head node will point to the tail as its previous. Circular linked lists allow for complete traversal starting from any node. They are useful for applications that function in a circular manner or loops.

What are the Advantages of a Linked List?


• Dynamic Size and Memory Usage

The size and memory space of a linked list changes as nodes are added and removed. When a node is added, more memory is allocated to the linked list, and when one is removed, memory is freed up. Thus, there is no way for a linked list to "waste" memory like how an array can. Furthermore, linked lists have no set capacity so it can handle data of unknown size by adapting.


• Fast Insertion and Deletion

In order to insert a node into a position within a linked list, all that needs to be done is alter the node at the previous position to point to the one being inserted. The node that the previous was originally pointing to will be set as the newly inserted node's next. Similarly for deletion, all that needs to be done is alter the node at the target's previous position to point to the target's next. There will be no node pointing to the target, effectively removing it from the linked list. A linked list performs insertion and deletion at the same speed, regardless of its size, making it very suitable for these functions.

What are the Disadvantages of a Linked List?


• Large Memory Usage

The nodes that make up a linked list take up quite a bit of memory space, as they have to store both data values and the memory addresses of their next nodes. This drawback gets worse in the case of doubly linked lists where two addresses need to be stored per node. As such, in situations where memory space is sparse, it may not be a good idea to use a linked list for data storage.


• Inefficient Accessing

In order to access a node at a specific position within a linked list, it is necessary to go through the list and past each node that comes before. As a result, accessing a node near the tail will take quite a bit of time. If the linked list contains many nodes, even more time will be needed. Circular doubly linked lists aren't as affected by this due to its ability to perform forward and backwards traversal from the head. However, the tradeoff is a sharp increase in memory usage for each node.


• Slow Traversal

Although the nodes in a linked list point to each other's locations in memory, they are stored non-contiguously. This means that their memory addresses are spread out, and nodes near each other within a linked list are usually far away from each other in memory. As a result, linked lists are not suited for traversing through since moving from one node to another cannot be done quickly.

Linked List Applications


A linked list can have various applications depending on its type. The standard singly linked lists should be used when insertions and deletions are frequently done to the stored data. Doubly linked lists should be used when backwards traversal is also needed, and circular linked lists allow for full traversal from anywhere within the data. Some examples of applications of this data structure are:


  • Music Playlist: In a playlist, songs are frequently added and removed. The ability to go to next or previous song is also a necessary feature. If the playlist needs to be repeated, the last song must be able to go back to the first. A circular doubly-linked list would be a good structure to story the songs in.

  • Browser Page History: Storing a browser's page history in a doubly linked list would allow users to go back to previously visited pages and return to current if they wished. Furthermore, when a user wants to clear parts of their browsing history, deletions needs to be done to the data, a function that linked lists specialize in.
